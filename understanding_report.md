# 深度強化學習 (HW3) 綜合理解報告

本報告綜合了 HW3-1 至 HW3-3 的實作與實驗結果，針對基礎 DQN、網路變體架構以及訓練工程框架進行深入分析。

---

## 1. 基礎 DQN 與 Experience Replay 的核心作用

在基礎 Deep Q-Network (DQN) 的訓練過程中，智能體 (Agent) 透過與環境互動所產生的連續經驗軌跡 $(S_t, A_t, R_t, S_{t+1})$ 具有極高的時間相關性 (Temporal Correlation)。若直接將這些序列資料輸入神經網路進行線上學習 (Online Learning)，會導致神經網路過度擬合於近期經驗，引發「災難性遺忘 (Catastrophic Forgetting)」，使得訓練極不穩定甚至發散。

**Experience Replay (經驗回放)** 機制透過建立一個容量固定的緩衝區 (Replay Buffer) 來儲存歷史互動經驗，完美解決了此問題：
*   **打破相關性 (Breaking Correlation)**：在訓練階段，模型改從緩衝區中進行**隨機抽樣 (Random Sampling)** 獲取小批量 (Mini-batch) 資料。此舉打破了資料間的時序相關性，使訓練樣本更符合獨立同分佈 (i.i.d.) 的假設，從而穩定神經網路的收斂。
*   **提升資料效率 (Data Efficiency)**：每一筆與環境互動的珍貴經驗皆可被保留並重複抽樣學習，在資料收集成本高的環境中，能大幅提升學習效率。

## 2. 變體分析：Double DQN 與 Dueling DQN 在隨機起點環境之表現

在 HW3-2 的 `player` 模式中（起點隨機），環境的不確定性增加，傳統 DQN 容易面臨 **Q 值高估 (Overestimation)** 的致命問題，導致策略在多個起始狀態下產生震盪與次優解。

*   **Double DQN (DDQN) 的改善**：
    基礎 DQN 在計算目標 Q 值時直接取最大值 $\max Q_{target}(S', a)$，這容易因網路初期的雜訊而選中被高估的動作。DDQN 透過**解耦「動作選擇」與「價值評估」**來修正此問題：由 Main Network 選擇下一個狀態的最佳動作，再交由 Target Network 評估該動作的真實價值。實驗對比顯示，DDQN 能有效抑制過度樂觀的預期，在隨機起點下收斂更為平穩，決策也更為精確。

*   **Dueling DQN 的優勢**：
    Dueling 架構在網路層級將 Q 值拆分為「狀態價值 (State Value, $V(s)$)」與「動作優勢 (Advantage, $A(s,a)$)」。在起點隨機的網格環境中，某些狀態的絕對價值（例如：遠離陷阱的安全空地）遠比採取特定動作更為重要。Dueling 架構允許模型在更新單一動作時，連帶更新該狀態的基準價值 $V(s)$。實驗證明，在面臨大量未知起始點時，Dueling DQN 能以極高的效率建立全局的價值認知，展現出三者中最強的**泛化能力 (Generalization)**。

## 3. 框架優勢：複雜任務中 PyTorch Lightning 與訓練技巧的必要性

進入 HW3-3 的 `random` 模式（所有物件位置皆隨機），狀態空間呈指數級別增長（超過 40,000 種組合），環境的稀疏獎勵與極高隨機性對訓練構成了嚴峻挑戰。在此情境下，引入現代深度學習框架與進階技巧至關重要：

*   **PyTorch Lightning 的模組化優勢**：
    在複雜的 RL 實驗中，PyTorch Lightning 透過封裝繁瑣的工程細節（如硬體調度、迴圈控制），使開發者能專注於演算法邏輯。其內建的 `CSVLogger` 與 `Trainer` 介面，確保了在動輒數萬步的長時間訓練中，Loss 變化與 Reward 趨勢能被穩定記錄與追蹤。

*   **Gradient Clipping (梯度裁剪) 的保護機制**：
    在全隨機配置下，Agent 偶爾會遭遇極端的初始狀態（例如：出生即緊鄰陷阱），這會產生巨大的誤差懲罰 (Loss Spike)，導致梯度爆炸並破壞已收斂的網路權重。引入 Gradient Clipping 能夠強制將異常梯度限制在安全範圍內（如 `norm=1.0`），是確保隨機環境長期訓練不崩潰的核心防護網。

*   **動態學習率 (LR Scheduling) 與正則化**：
    配合 AdamW 優化器的權重衰減 (Weight Decay) 來預防模型對特定地圖配置過擬合；同時搭配 `StepLR` 調度器，使模型在探索初期擁有較大的步長以快速理解廣袤的狀態空間，並在訓練後期自動降低學習率進行精細的策略微調。這些工程技巧的疊加，是讓深度強化學習從「理論可行」跨越到「實作收斂」的關鍵所在。
